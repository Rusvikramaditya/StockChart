Phase 3: Pattern Overlay Cleanup IN PROGRESS chart_annotations.js

Design principle

The large centered title already tells the viewer what pattern it is. The overlay should only draw geometric shapes — no text labels on the canvas body. This matches the reference images exactly (reference chart 1: just a diagonal line; reference chart 2: just two horizontal bands).

Exception: "Neckline" label in IHS. Keep this single label — it marks the exact entry trigger price and is actionable info that isn't communicated by the geometry alone.

Phase 4: Bottom Insight Text + Scale Margin Fix IN PROGRESS chart_renderer.js

Bottom insight text

A centered HTML overlay div at the bottom of the chart, above the TradingView attribution. Displays upside %, R:R, or custom text.

// Positioned absolute bottom, centered
var insightEl = document.createElement('div');
insightEl.style.cssText = [
  'position:absolute',
  'bottom:28px',          // above TradingView attribution
  'left:0',
  'right:90px',           // leave room for price scale
  'text-align:center',
  'pointer-events:none',
  'z-index:4',
  'user-select:none',
].join(';');

// Content: "Upside +17.4%  ·  R:R 2.7:1"
var insightParts = [];
if (tp.upside_pct != null) insightParts.push('Upside +' + tp.upside_pct.toFixed(1) + '%');
if (tp.reward_risk != null) insightParts.push('R:R ' + tp.reward_risk.toFixed(1) + ':1');
var insightText = insightParts.join('  ·  ');

insightEl.innerHTML = '<div style="font-size:' + (compact ? '11px' : '14px') + ';' +
  'font-weight:700;color:rgba(0,0,0,0.60);' +
  'font-family:Inter,Segoe UI,Arial,sans-serif;">' +
  _esc(insightText) + '</div>';
container.appendChild(insightEl);

Anti-Patterns to Avoid

DO NOT use entry price as neckline fallback in IHS — it places the neckline below the pattern (already fixed).
DO NOT draw the handle as a line from handle_start to right_rim — it draws backward (already fixed).
DO NOT use a fixed pixel offset (e.g. trough.y + 24) for bezier control points — use the midpoint formula (already fixed).
DO NOT use relativeChartIndex fallback that maps out-of-range indices to candle array index — causes wrong dot placement (already fixed).
DO NOT hardcode text like "Upside +17.4%" inside fill rectangles — move to labeled badges above boxes.
DO NOT ask for confirmation mid-task — fix all items in scope and run tests.
